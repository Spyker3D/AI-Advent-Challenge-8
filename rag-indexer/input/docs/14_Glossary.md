# 14_Glossary

# Глоссарий Android AI Assistant

## Назначение
Этот документ содержит основные термины курса. Он полезен как справочник и хорошо подходит в качестве базы знаний для RAG, поскольку каждое понятие имеет отдельное краткое определение.

## Термины

### API
Интерфейс программирования приложений. Набор методов и правил, позволяющий приложениям взаимодействовать друг с другом.

### Coroutine
Лёгкая асинхронная задача Kotlin, позволяющая выполнять длительные операции без блокировки главного потока.

### Flow
Асинхронный поток данных из Kotlin Coroutines, используемый для реактивной передачи значений.

### Composable
Функция Jetpack Compose, помеченная @Composable, которая описывает пользовательский интерфейс.

### Repository
Слой доступа к данным, скрывающий детали работы с сетью, базой данных и кэшем.

### DI
Dependency Injection — способ передачи зависимостей объектам извне для уменьшения связанности кода.

### Embedding
Векторное представление текста, позволяющее сравнивать смысловую близость документов.

### Chunk
Небольшой фрагмент документа, используемый при индексировании и поиске в RAG.

### Vector
Числовой массив, описывающий смысл текста в многомерном пространстве.

### Prompt
Инструкция или запрос, отправляемый языковой модели.

### Tool
Внешняя функция или сервис, который модель может вызвать для выполнения действия.

### MCP
Model Context Protocol — открытый протокол подключения AI к инструментам и источникам данных.

### LLM
Large Language Model — большая языковая модель, генерирующая и анализирующая текст.

### RAG
Retrieval-Augmented Generation — подход, объединяющий поиск релевантных данных и генерацию ответа.

### Streaming
Постепенная передача ответа модели по мере генерации, без ожидания полного результата.

### JSON
Текстовый формат обмена данными, широко используемый в REST API.

### DTO
Data Transfer Object — объект для передачи данных между слоями приложения или по сети.

### Domain
Слой бизнес-логики приложения, независимый от Android Framework.

### UI State
Полное описание текущего состояния пользовательского интерфейса.

## Рекомендации для RAG
- Индексируйте каждое определение как отдельный chunk.
- Добавляйте название термина в метаданные.
- Используйте этот файл как словарь при поиске контекста.
- Регулярно дополняйте глоссарий новыми понятиями проекта.

# Итоги
Глоссарий служит единым источником определений для разработчиков, документации и RAG-системы. Наличие коротких независимых определений повышает качество семантического поиска и уменьшает неоднозначность ответов AI.
