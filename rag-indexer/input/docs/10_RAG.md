## Что такое RAG

Retrieval-Augmented Generation (RAG) — это подход, при котором языковая модель получает не только запрос пользователя, но и дополнительный контекст, найденный в базе знаний.

Вместо ответа только на основе обучающих данных модель использует актуальную информацию, найденную непосредственно перед генерацией ответа.

---

## Почему RAG лучше обычного LLM

Без RAG модель:

- может "галлюцинировать";
- не знает новых данных после окончания обучения;
- не имеет доступа к корпоративным документам.

С RAG модель:

- отвечает на основе найденных документов;
- использует свежую информацию;
- может работать с внутренней документацией компании.

---

## Архитектура RAG

```
Пользователь
      │
      ▼
Запрос
      │
      ▼
Поиск релевантных документов
      │
      ▼
Векторная база знаний
      │
      ▼
Найденный контекст
      │
      ▼
LLM
      │
      ▼
Ответ
```

---

## Основные этапы работы

1. Пользователь отправляет вопрос.
2. Вопрос преобразуется в embedding.
3. Выполняется поиск похожих документов.
4. Найденные фрагменты добавляются в prompt.
5. LLM формирует окончательный ответ.

---

## Компоненты системы

### Embedding Model

Преобразует текст в числовой вектор.

### Vector Database

Хранит embeddings и выполняет поиск по семантической близости.

### Retriever

Находит наиболее релевантные документы.

### Generator

Использует найденный контекст и создаёт ответ.

---

## RAG в Android AI Assistant

Типичный сценарий:

- пользователь задаёт вопрос;
- приложение обращается к базе знаний;
- релевантные документы передаются модели;
- AI отвечает с учётом найденного контекста.

Такой подход подходит для документации, FAQ, внутренних инструкций и корпоративных знаний.

---

## Ограничения

- качество ответа зависит от качества поиска;
- необходима предварительная индексация документов;
- слишком большой контекст увеличивает стоимость и время ответа.

---

## Best Practices

- Разбивайте документы на небольшие смысловые фрагменты.
- Используйте качественную embedding-модель.
- Ограничивайте количество передаваемых фрагментов.
- Регулярно обновляйте индекс базы знаний.
- Комбинируйте RAG с MCP для доступа к внешним данным и инструментам.

---

# Итоги

RAG значительно повышает качество ответов AI за счёт использования актуального контекста. В Android AI Assistant этот подход позволяет строить интеллектуальные помощники, работающие с документацией, базами знаний и другими источниками информации без дополнительного обучения модели.
