## Что такое MCP

Model Context Protocol (MCP) — это открытый протокол, который позволяет AI-моделям безопасно взаимодействовать с внешними источниками данных и инструментами через единый стандарт.

Вместо написания отдельных интеграций для каждого сервиса AI-клиент может работать с любым MCP-сервером.

---

## Зачем нужен MCP

MCP решает несколько задач:

- подключение внешних инструментов;
- доступ к базам знаний;
- выполнение действий через API;
- единый способ обмена контекстом между AI и приложением.

## Клиент и сервер MCP

**MCP Client** находится в приложении и отправляет запросы.

**MCP Server** предоставляет инструменты, ресурсы и инструкции, которые может использовать AI.

---

## Инструменты (Tools)

Инструменты позволяют модели выполнять действия.

Примеры:

- поиск документов;
- отправка HTTP-запросов;
- работа с календарём;
- чтение файлов;
- выполнение SQL-запросов.

---

## Resources

Resources предоставляют данные только для чтения.

Например:

- документация;
- локальные файлы;
- PDF;
- статьи;
- базы знаний.

Модель получает эти данные как дополнительный контекст.

---

## Prompts

MCP-сервер может публиковать готовые шаблоны запросов (Prompts), которые помогают стандартизировать взаимодействие с моделью.

---

## Использование в Android AI Assistant

Типичный сценарий:

1. Пользователь задаёт вопрос.
2. AI определяет необходимость использования инструмента.
3. MCP Client обращается к MCP Server.
4. Сервер возвращает данные.
5. AI формирует окончательный ответ.

Такой подход уменьшает количество «галлюцинаций» модели и позволяет работать с актуальной информацией.

---

## Best Practices

- Делите инструменты по зонам ответственности.
- Не передавайте модели лишние данные.
- Ограничивайте доступ к чувствительной информации.
- Логируйте обращения к MCP-серверу.
- Используйте аутентификацию и шифрование соединений.

---

# Итоги

MCP становится стандартным способом подключения AI к внешним данным и сервисам. Использование MCP в Android AI Assistant позволяет сделать приложение расширяемым, безопасным и независимым от конкретных поставщиков инструментов.
